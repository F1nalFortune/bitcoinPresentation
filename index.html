A Peer-to-Peer Electronic Cash System


Created by: Brian Phelps

What is Bitcoin


A purely peer-to-peer version of electronic cash that allows online payments to be sent directly from one party to another without going through a trusted third party.

Double Spending Problem

Back in the early 1990s, developers and cryptographers were trying to solve the double-spend problem. Digital currencies are just another form of digital files.
Bitcoin is the first protocol to solve this problem.

Timestamp Server

Need a way to publically publish time stamps. Each timestamp includes the previous timestamp in its hash, forming a chain, with each additional timestamp reinforcing the ones before it.

Proof of Work

In order to add a new block, you need to spend a lot of hash power. Therefore, connection between blocks should not be easy to make.
Connection requires a lot of hash power, electricity, and it takes time.

Network

Bitcoin network is comprised many nodes. Currently, nodes do not mine.

Incentive

Cost for you to mine a block. If you're able to successfully mine a block, you will get a block reward. First transaction on block goes to the miner. Next transaction comes from other wallets. First transactions on block are called coinbase transactions, no relation to the company coinbase or the exchange.

Reclaiming Disk Space

You do not need to store all of the hashes on the chain, you can store a hash of the hashes. This is what is meant by a Merkle Tree described in the white paper.

Simplified Payment Verification

SPV only uses portion of network by only using a portion of the network. It is possible to verify transactions without running a full network node. A user needs only to keep a copy of block headers.
